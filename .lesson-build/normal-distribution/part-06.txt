5%.</strong> 370 = 350 + 2&times;10 = &mu; + 2&sigma;. About 95% of bottles fall within
        &plusmn;2&sigma;, leaving 5% outside. The curve is symmetric, so half of that 5% &mdash; about 2.5% &mdash; is
        above +2&sigma;.</p>
        <p><strong>3. Almost never &mdash; about 0.1&ndash;0.15% of bottles.</strong> 320 = 350 &minus; 3&times;10 = &mu; &minus; 3&sigma;.
        About 99.7% of bottles fall within &plusmn;3&sigma;, leaving 0.3% outside, split between two tails.
        Below &minus;3&sigma; is roughly 0.15% &mdash; about 1 or 2 bottles in every 1,000.</p>
        <p><strong>The reasoning pattern:</strong> convert each boundary into &ldquo;how many &sigma; from &mu;?&rdquo;,
        then read the proportion straight off the 68&ndash;95&ndash;99.7 rule, remembering to halve any leftover
        percentage when the question asks about one tail only.</p>
      </div>
    </div>
  </div>
</section>

<section id="summary" aria-labelledby="summary-h">
  <h2 id="summary-h">Final summary</h2>
  <h3>Key takeaways</h3>
  <ul class="takeaways">
    <li>The normal distribution is a symmetric, single-peaked model where values cluster around the mean.</li>
    <li>Two numbers describe it completely: the mean &mu; (location) and the standard deviation &sigma; (spread).</li>
    <li>The empirical rule: about 68% of values fall within &plusmn;1&sigma;, 95% within &plusmn;2&sigma;, 99.7% within &plusmn;3&sigma;.</li>
    <li>A z-score, z = (x &minus; &mu;) / &sigma;, converts any value into &ldquo;standard deviations from the mean&rdquo; so different measurements can be compared fairly.</li>
    <li>|z| beyond 2 is unusual; |z| beyond 3 is rare and usually worth investigating.</li>
    <li>Software gives the area <em>below</em> a value by default &mdash; subtract from 100% when you need the area above.</li>
    <li>Always plot your data first: skewed, bounded, or two-peaked data breaks the normal model.</li>
  </ul>
  <h3>Decision rule</h3>
  <div class="panel callout">
    <p>If the histogram shows <strong>one peak and rough symmetry</strong>, use the normal model:
    boundaries at whole multiples of &sigma;  empirical rule; anything else  z-score plus a table or
    software. If the histogram is <strong>skewed, bounded, or two-peaked</strong>, do not use the normal model.</p>
  </div>
  <div class="remember">Remember this: subtract the mean, divide by &sigma; &mdash; then let 68&ndash;95&ndash;99.7 do the talking.</div>

    to solidify the building blocks) or <em>Confidence Intervals for Beginners</em> (to see the normal curve

  // Normal probability density function
  function normalPdf(x, mu, sigma) {
    var zz = (x - mu) / sigma;
    return Math.exp(-0.5 * zz * zz) / (sigma * Math.sqrt(2 * Math.PI));
  }

  // Standard normal cumulative distribution, Phi(z).
  // Zelen & Severo (1964) polynomial approximation, |error| < 7.5e-8.
  function normalCdf(z) {
    var t, d, p;
    t = 1 / (1 + 0.2316419 * Math.abs(z));
    d = 0.3989422804014327 * Math.exp(-0.5 * z * z);
    p = d * t * (0.319381530 + t * (-0.356563782 + t * (1.781477937 + t * (-1.821255978 + t * 1.330274429))));
    return z >= 0 ? 1 - p : p;
  }

  function fmt(n, dp) {

  }
